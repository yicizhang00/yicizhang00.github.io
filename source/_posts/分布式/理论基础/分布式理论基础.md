# 向量时钟
Lamport在经典论文《Time, Clocks, and the Ordering of Events in a Distributed System》中解释了向量时钟的概念。根据相对论，每台机器没有一个统一的时间，因此在分布式系统中采用因果时序（causal ordering）来建模事件之间的关系。

向量时钟（Vector Clock）是一种用于捕获分布式系统中事件因果关系的机制。它扩展了Lamport逻辑时钟，能够区分并发事件和因果相关事件。

Lamport认为判定一个系统是不是分布式不是根据距离而是延时，如果消息传输延迟与单个进程内事件发生的时间间隔相比不可忽略，那么该系统就是分布式的。单台服务器也可以被视为一个分布式系统，其中中央控制单元（CPU）、存储单元（Memory）和输入输出通道（I/O Channels）都是独立的进程。

传统的判断一个事件a发生于b之前，可以采用事件a发生的时间早于b，如果一个系统需要依据于某种规范，那么该规范必须根据系统内**可观测**的事件来给出，传统的规范依赖于**物理时间**，但是这类时间的物理时钟并非完美。

> **物理时钟**：使用实际的墙上时间（wall-clock time），如NTP同步的时钟。

然而，在分布式系统中，物理时钟存在以下问题：
- 时钟漂移（clock drift）：时钟运行速度不一致。
- 时钟偏差（clock skew）：时钟之间的初始差异。根据相对论，两台机器中的时间不可能是一致的。
- 网络延迟导致的不确定性。

因此，物理时钟不能可靠地用于确定因果关系。

 > **偏序关系**：在分布式系统中，事件之间存在**偏序关系**（Partial Order），也称为“happens-before”关系，由Lamport定义。记作“→”。

- 如果事件a和b在同一进程中，且a在b之前发生，则a → b。
- 如果事件a是一个进程发送消息，b是另一个进程接收该消息，则a → b。
- 传递性：如果a → b且b → c，则a → c。

偏序关系不能比较所有事件对，有些事件是并发的（concurrent），即既没有a → b也没有b → a。

> **全序关系**（Total Order）：是对偏序关系的扩展，要求任意两个事件都能比较大小。

在分布式系统中，可以通过添加进程ID或时间戳来将偏序转换为全序。例如，使用Lamport时钟加上进程ID。

全序关系对于实现一致性协议很重要，如总序广播（Total Order Broadcast）。

> **时钟条件** 如果a → b，那么C(a) < C(b) ，但是该条件的逆命题不成立，因为两个并发事件虽然有C(a),C(b)，他们的值也可以比较，但是a和b实际上不能判断因果关系。

如果满足以下两个条件，则时钟条件成立：
C1：如果 $a$ 和 $b$ 是进程 $P_i$ 中的事件，且 $a$ 在 $b$ 之前出现，则 $C_i(a) < C_i(b)$。
C2：如果 $a$ 是进程 $P_i$ 发送消息的事件，$b$ 是进程 $P_j$ 接收该消息的事件，则 $C_i(a) < C_j(b)$。

> **逻辑时钟**（Logical Clock）：由Lamport提出，不依赖物理时间，而是为每个事件分配一个逻辑时间戳。

每个进程维护一个计数器C，当发生本地事件或发送/接收消息时，更新C。

- 本地事件：C = C + 1
- 发送消息：C = C + 1，附上时间戳
- 接收消息：C = max(C, 接收的时间戳) + 1

逻辑时钟可以保持happens-before关系，但无法区分并发事件。

## 向量时钟

**向量时钟**（Vector Clock）是逻辑时钟的扩展，每个进程维护一个向量，向量的长度等于进程数量。

每个进程Pi维护向量VCi[1..n]，其中VCi[i]是进程Pi对进程Pj的逻辑时钟。

更新规则：
- 本地事件：VCi[i] = VCi[i] + 1
- 发送消息：VCi[i] = VCi[i] + 1，将VCi附上消息
- 接收消息：对于每个j，VCi[j] = max(VCi[j], VCj[j])，然后VCi[i] = VCi[i] + 1

向量时钟可以精确捕获因果关系：
- 如果VCa < VCb（向量比较），则a → b
- 如果VCa 和 VCb 不可比较，则a 和 b 并发

向量时钟用于检测因果一致性、版本向量等应用。
